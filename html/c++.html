1.1 변수

· 자료형 변수명(이름) 형태, RAM의 메모리 블록에 값을 할당하여 사용하기 위한 것

1.1.3 변수의 초기화

· C++은 변수를 선언하자마자 초기화해야한다. 값을 지정(즉 선언과 동시에 초기화)하지 않으면

1.1.3 변수의 분류

"(추후 추가) 분류 그림, 표"
· 변수를 선언하는 순간, 변수는 저장공간 위치, 생명주기, 스코프(사용범위), 링크를 가지게된다.

1.1.3.1 저장공간 위치
- 데이터 영역

· 프로그램의 시작부터 종료까지 점유되는 영역이다.
· 전역 변수, 정적 변수가 저장되는 영역이다.

- 스택 영역

· 프로그램이 자동으로 사용하는 임시 공간이다.
· 지역 변수, 매개 변수가 저장되는 영역이다.
· 메모리 주소가 내림차순으로 저장되며 후입선출(LIFO)구조이다.
· 함수 호출 정보도 저장되는데 스택 프레임이라한다.

- 힙 영역

· 사용자가 직접 할당한 동적 변수가 저장되는 영역이다.

1.1.3.1 생명 주기(life-cycle)
- 자동 생명 주기(= 자동 변수)

· 지역변수, 매개 변수가 있다. 즉, 런타임의 특정 시점에 자동으로 생성, 소멸한다.

- 정적 생명 주기(= 정적 변수)

· 전역변수, 정적 변수가 있다. 즉, 런타임 시작 생성, 런타임 종료 소멸한다.
· 모든 정적 변수는 초기화되지 않으면 비트가 0, 즉 0으로 세팅된다. 이를 제로 초기화(zero-initialized)라고 한다.

1.1.3.2 사용 범위(scope)

· 변수명이 하나의 파일 안에서 얼마나 널리 알려지는가이다.

- 블록 사용 범위(= 지역 변수 = 지역 사용 범위)

· 블록(중괄호로 묶인 연속된 구문)안에서만 알려진다.
· 자동 변수는 블록 사용범위이다.

- 전역 사용 범위(= 전역 변수 = 외부 변수 = 파일 사용 범위)

· 변수명이 선언된 시점부터 그 아래 모든 줄에 알려진다.

1.1.3.3 링크(Linkage)

· 서로 다른 파일들이 이름을 공유하는 것이다.
· 자동변수는 공유되지 않기 때문에 링크를 가지지 않는다.

- 외부 링크(External Linkage)

· 변수명을 여러 파일이 공유할 수 있다.

- 내부 링크(Internal Linkage)

· 그 변수가 선언된 한 파일 내부에서만 공유할 수 있다.

· 우리는 변수를 무조건 함수 내부(지역) 혹은 외부(전역)에서 선언한다.
· 즉, 지역 변수, 전역 변수의 선언 그리고 그 앞에 붙는 키워드에 따라서 어떤 변수가 생성되는지를 파악하는것이 이해가 쉬울 것이다.

- 지역 변수           (자동 생명주기(= 자동 변수), 지역 스코프(= 지역 변수), 링크 없음)

· 앞서 설명했다시피, 지역 변수는 링크가 무조건 없다. 왜냐하면 언제 생성되고 소멸될지 모르기 때문에
    물리적 관점에서 링크를 부여할 수가 없다.

- 레지스터 변수   (자동 생명 주기(= 자동 변수), 지역 스코프(= 지역 변수), 링크 없음)

· 생명주기, 스코프, 링크는 지역 변수와 같다.
ㆍregister 키워드는 C에서는 자동 변수를 CPU의 register에 저장하는 기능이었다.
ㆍregister 키워드는 C++11이전 버전에서는 H/W와 컴파일러가 매우 정밀하게 개발된 경우를 제외하고는
   "자주 사용되는 변수" 혹은 "특별한 조치가 필요한 변수"의 의미로 사용되었다.
ㆍregister 키워드는 C++11이후 버전에서는 자동 변수임을 명시하는데 사용하며,
   오로지 자동 변수만 사용하는 경우에는 사용자가 전역 변수와 동일한 이름을 지닌 자동 변수(지역 변수)를 구분짓는 의미로 쓰이기도 한다.
ㆍregister키워드의 의미를 대체 하기 위해 auto키워드를 쓸 수 있지만,
   기존 코드가 auto 키워드를 인식하지 못하는 경우를 대비하여 C++11이후 버전에서도 register 키워드가 계승되었다.

- 정적 지역 변수  (정적 생명 주기(= 정적 변수), 지역 스코프(= 지역 변수), 링크 없음)

· 지역 변수 앞에 static 키워드를 붙이면 컴파일 타임에 컴파일러가 자동 생명 주기를 정적 생명 주기로 변경한다.

- 전역 변수           (정적 생명 주기(= 정적 변수), 전역 스코프(= 전역 변수), 외부 링크)

· 디폴트가 외부 링크(extern)이다.
· 다른 외부 파일에서 사용시 외부 파일에 선언해야한다.
· 단일 정의 원칙(odr)에 의해 변수는 단 한번 선언되야하므로,(하나의 전역 변수는 단 하나의 전역 변수 정의, 하나의 지역 변수는 단 하나의 지역 변수 정의)
   외부 파일 선언시 참조 선언(선언 앞에 extern추가, 초기화 하지 않음 유의!)을 실시한다.
· 🔗전역 변수에 붙는 extern 키워드의 2가지 의미
1. 외부 링크를 가지는 전역 변수의 명시적 표현 (초기화 한 경우)
2. 외부 다른 파일에 정의된 전역 변수에대한 전방 선언을 의미한다. (초기화 하지 않은 경우)
· 전방 선언의 경우 함수 외부에서 쓰면 그 파일 전체에, 함수 내부에서 쓰면 해당 함수 블록에서만 알려진다. · 전역 변수와 지역 변수는 다른 버전이라서 단일 정의 원칙에 저촉되지 않는다.
· 하지만 지역 스코프 내에서 전역 변수가 가려지는 상황에서는 사용 범위 결정 연산자(::)를 앞에 붙일 경우 지역 변수가 아닌 전역 변수로 인식된다.

1.2.2 rvalue 참조형

1.2.2.1 lvalue, rvalue 란?

· C부터 내려온 것으로 일반적으로 대입 연산자(=)의 피연산자 중 좌항을 lvalue 우항을 rvalue라고 한다.
· 틀린이야기는 아니나, 다른 관점에서 볼 필요가 있다.

- C++에서 lvalue, rvalue란

· lvalue는 메모리상에서 지속적인 주소를 가진 값이다. 즉, 변수인데 표현식(expression) 이후에도 없어지지 않는다.
· rvalue는 지속적인 메모리 주소와 관계 없는 값이다. 즉, 숫자나 표현식(2, x + 2 등...), 표현식 이후에는 사라지는 값이다.
