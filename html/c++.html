1. 변수 관련

1.1 변수

· 자료형 변수명(이름) 형태, RAM의 메모리 블록에 값을 할당하여 사용하기 위한 것
· 변수명(이름)은 변수로 지정된 메모리 블록에 붙여주는 이름
· 자료형은 메모리 블록의 비트를 어떤 값으롤 해석할 건지 결정
"(추후 추가)램의 메모리 블록 그림에 비트 변수명 자료형 어떻게 해석되는지 그림 위치"

1.1.3 변수의 초기화

· C++은 변수를 선언하자마자 초기화해야한다. 값을 지정(즉 선언과 동시에 초기화)하지 않으면
   메모리 블록에 설정된 임의의 비트들을 자료형에 의거해 해석하므로 이상한 값이 할당된다.
· 초기화를 하지 않으면 대부분의 컴파일러가 경고를 출력한다. · 변수의 초기화 방법에는 3가지가 있다.

1.1.3 변수의 분류

"(추후 추가) 분류 그림, 표"
· 변수를 선언하는 순간, 변수는 저장공간 위치, 생명주기, 스코프(사용범위), 링크를 가지게된다.
· 변수는 몇가지 방법으로 분류할 수 있지만, 우리가 코드를 작성함에 있어서 실제로 보는 선언 위치에 따라 분류하고 이해해보자.
· 지역 변수는 자동 생명주기를 가지기 때문에 스택 영역에 저장된다 생성과 소멸이 함수나 스코프에 들어갈 때 일어나기 때문이다.
· 첨언하자면, 지역 변수의 저장과 같은 원리로 함수의 주소 또한 실행과 종료에 따라야하므로 스택 프레임이라는 곳에 저장된다.
· 전역 변수는 정적 생명 주기를 가지므로 스택과 같은 구조가 필요 없이 그냥 데이터 영역에 저장만 하면 된다.

1.1.3.1 지역 변수
- 지역 변수

· 스택 영역, 지역 스코프, 자동 생명주기, 링크 없음

- static 지역 변수

· 데이터 영역, 지역 스코프, 정적 생명주기, 링크 없음

- register 변수

· CPU의 레지스터, 지역 스코프, 자동 생명주기, 링크 없음
ㆍregister 키워드는 C에서는 자동 변수를 CPU의 register에 저장하는 기능이었다.
ㆍregister 키워드는 C++11이전 버전에서는 H/W와 컴파일러가 매우 정밀하게 개발된 경우를 제외하고는
ㆍ"자주 사용되는 변수" 혹은 "특별한 조치가 필요한 변수"의 의미로 사용되었다.
ㆍregister 키워드는 C++11이후 버전에서는 자동 변수임을 명시하는데 사용하며,
ㆍ오로지 자동 변수만 사용하는 경우에는 사용자가 전역 변수와 동일한 이름을 지닌 자동 변수(지역 변수)를 구분짓는 의미로 쓰이기도 한다.
ㆍregister키워드의 의미를 대체 하기 위해 auto키워드를 쓸 수 있지만,
ㆍ기존 코드가 auto 키워드를 인식하지 못하는 경우를 대비하여 C++11이후 버전에서도 register 키워드가 계승되었다.

1.1.3.2 전역 변수
- 전역 변수

· 데이터 영역, 전역 스코프, 정적 생명주기, 외부 링크

- extern 전역 변수

· 초기화 실시여부에 따라 2가지로 나뉨
· 초기화 O, 정의
· 초기화 X, 참조 선언, 외부 파일에 선언된 전역 변수를 가져와 쓰겠다라는 의미(단일 정의 원칙: '한 변수는 한 번만 선언' 준수위함임)
· 즉, 여러 파일에 같은 이름의 전역 변수가 여러개 있다면, 반드시 단 하나는 선언이어야하고, 나머지들은 참조 선언이어야 한다.
· 여러 파일에 같은 이름의 전역 변수들이 모두 선언이면 odr(단일 정의 원칙)위배이고, 반대로 모두 참조 선언이면 참조할 변수가 없는 오류가 발생한다.
· 참조 선언은 말 그대로 어떤 전역 변수를 참조하는 변수이므로 메모리 블록이 할당되지 않는다.

- static 전역 변수

· 데이터 영역, 전역 스코프, 정적 생명 주기, 내부 링크
· static은 '정적'임을 나타내는데, 전역 변수는 이미 정적이다.
· static이 전역 변수에 붙으면 이미 정적인 생명 주기 대신 링크를 외부 링크에서 내부 링크로 바꾼다고 이해하자.

- const 전역 변수

· 데이터 영역, 전역 스코프, 정적 생명 주기, 내부 링크
· 즉, 전역 변수가 상수화 되면, 외부 링크가 내부 링크로 바뀜

1.1.4 전역 변수 파고들기

1.1.4.1 같은 이름의 전역 변수 지역 변수

· 마치 단일 정의 원칙에 위배될 것 같지만, 그렇지 않다.
· 다만 지역 내에서는 지역 변수가 전역 변수를 이긴다. 이를 전역 변수의 이름을 지역 변수가 가린다.
· C++에서는 사용 범위 결정 연산자(::)로 지역 내에서 같은 이름의 전역 변수를 부를 수 있다.

1.2.2 rvalue 참조형

1.2.2.1 lvalue, rvalue 란?

· C부터 내려온 것으로 일반적으로 대입 연산자(=)의 피연산자 중 좌항을 lvalue 우항을 rvalue라고 한다.
· 틀린이야기는 아니나, 다른 관점에서 볼 필요가 있다.

- C++에서 lvalue, rvalue란

· lvalue는 메모리상에서 지속적인 주소를 가진 값이다. 즉, 변수인데 표현식(expression) 이후에도 없어지지 않는다.
· rvalue는 지속적인 메모리 주소와 관계 없는 값이다. 즉, 숫자나 표현식(2, x + 2 등...), 표현식 이후에는 사라지는 값이다.
