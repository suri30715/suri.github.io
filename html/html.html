· 웹 브라우저에 보여질 문서를 만드는 언어이다. Hyper Text(문서를 연결해주는 링크)기능을 가진 문서이다. (웹 문서)
· 웹 문서(웹 페이지란) WWW상의 모든 문서들을 뜻한다.
· Hyper Text Markup Language의 약자
· Hyper Text란 '문서를 서로 연결해주는 링크',
· Markup 이란 '표시한다' 즉, '웹 브라우저에 내용을 보여주는 텍스트, 이미지, 영상등의 위치를 표시한다. '라는 의미.
· 프로그래밍 언어가 아닌 마크업 언어
· 웹 페이지(= HTML 문서)는 HTML 태그들로 구성
· 태그(tag): 웹 브라우저가 문서의 어느 부분이 무엇을 의미하는지 구별하도록 해주는 꼬리표이다.
· 시멘틱 태그(semantic tag): HTML 구조 설계에 있어 태그에 의미(semantic)를 부여하여, 웹 페이지 구조를 쉽게 파악하도록 한 태그(예: <header>, <nav>, <article> 등)
· 빈 태그(empty tag): 닫힌 태그가 없는 태그를 말한다. (예: <br>, <img> 등)
· HTML5부터 속성값에 따옴표를 강제하지 않지만,
· 속성값은 반드시 따옴표로 감싸야 왜냐하면 속성 값에 띄어쓰기가 있으면 따옴표로 감싸야 정확하게 인식한다.
· 인터넷의 발전에 따라 HTML 버전도 업그레이드 중이다. 최신 HTML 정보는 위 W3C링크에서 확인 가능하다.

· HTML 기본 문서 구조는 IDE에서 <!를 친 후 Enter키를 누르면 자동 완성된다.
· <!DOCTYPE html>: 문서 유형(Document Type)을 지정하는 태그 (DTD; Document Type Definition)
· <html>: 문서의 루트(root)요소를 정의한다. · <html>의 lang 속성: 검색 엔진에서 특정언어로 제한해 검색할 때 필요한 것,
· 검색 결과에서 한국어로 된 문서로 범위를 제한하면 lang = "ko"인 문서를 우선 검색함.
· <head>: head요소들은 대부분 웹 브라우저에 보이지 않음, 웹 브라우저가 알아야할 정보를 요소로 가짐.
· <meta>: 메타데이터 (데이터에 관한 데이터; 책의 가격, 쪽수, 지은이를 '책의 메타데이터'라고 함.)
· 즉, 웹 문서의 meta태그도 웹 문서와 관련된 정보를 정의한다.
· 웹 브라우저는 영어가 디폴트이므로, 한글을 사용하려면, charset = "UTF-8" 즉 UTF-8문자세트를 사용한다고 해야함
· 문자셋(character set): 대표적인 것은 ↓
· ASCII (127개의 영문자, 숫자 최초의 문자셋)
· ANSI (윈도우즈 문자셋 256개 지원)
· ISO-8859-1 (HTML4의 기본 문자 셋 256개 지원)
· UTF-8 (세상의 거의 모든 문자를 표현할수 있는 유니코드 문자를 지원하는 HTML5의 기본 문자셋)
· <title>: 웹 문서의 제목을 표시한다. 요소를 입력하면 웹 브라우저의 제목 표시줄에 나타난다.
· <body>: 실제 웹 브라우저에 표시될 내용을 정의한다. body요소에서 대부분의 태그를 사용한다.

블록과 인라인

· HTML의 모든 요소는 display속성을 가짐
· display속성은 해당 요소가 웹 브라우저에서 어떻게 보여지는가를 의미함.
· HTML의 대부분 요소는 block과 inline중 하나
· block: 항상 새로운 라인에서 시작되며, 그 라인의 모든 너비를 차지, 블록 요소라고도함 (p, div, h, ul, ol, form이 대표적)
· inline: 새로운 라인에서 시작하지 않음, 해당 요소의 내용 만큼 차지, 인라인 요소라고도함 (spam, a, img가 대표적) · div는 대표적 블록 요소
· span은 대표적 인라인 요소
· CSS에서 display속성으로 잘 사용하므로 숙지해야함.
